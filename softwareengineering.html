  1. Requirements analysis
  2. Design

Software Life Cycle: Development phase

  1. specifies what services the proposed system will provide, to identify any conditions (time constraints, security) and to define how the outside world will interact with the system
  2. creates a plan for the construction of the proposed system
  3. actual writing of programs, creation of data files, and development of databases
  4. equated with the process of debugging programs and confirming that the final software product was compatible with the software requirements specification

Software Testing:

  1. Glass box testing
  2. Black box testing
  3. Beta testing
  4. Alpha testing

Software Testing

  1. software tester is aware of the interior structure of the software and uses this knowledge when designing the test
  2. tests that do no rely on knowledge of the software's interior composition
  3. falls within the black-box; a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market
  4. similar testing (beta) performed at the developer's site

Documentation:

  1. User documentation
  2. System documentation
  3. Technical documentation

Documentation:

  1. explain the features of the software and describe how to use them
  2. describes the software's internal composition so that the software can be maintained later in its life cycle
  3. describes how a software system should be installed and serviced (such as adjusting operating paramets, installing updates, and reporting problems back to the software's developer)
